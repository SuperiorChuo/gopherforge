# 系统配置与运营

system 服务承载"轻配置态"运营能力，核心理念：**需要配置的参数一律进数据库热生效，环境变量只兜底**。

## 系统设置（DB 热配置）

`system_settings` 表按 `分组.键` 存 JSON 配置，控制台「系统设置」页元数据驱动渲染表单，保存即 upsert：

- `security.policy`：密码策略、登录限流阈值
- `ai.provider` 等：各服务照同一范式接入（TTL 缓存 + env 兜底，改配置约 30 秒热生效，无需重启）

新配置项只需：后端加一个带 TTL 缓存的 reader + 前端 FIELD_SCHEMAS 加一段字段声明。

## 字典管理

字典类型 + 字典项二级结构，业务下拉/标签的统一数据源。

## 通知公告

发布/下架管理，前端登录后拉取生效公告展示。

## 短信管理

渠道（provider 可插拔：debug/阿里云/腾讯云）+ 模板 + 发送日志三件套；debug 渠道本地开发零依赖联调。

## 错误码管理

错误码表 + 管理页，**在线改文案 30 秒热生效**（30s TTL），不用为改一句提示语重新发版。

## 在线用户

Redis 会话登记，可查看当前在线并强制下线。
